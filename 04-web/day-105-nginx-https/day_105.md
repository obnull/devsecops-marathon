# Stage 01: Linux & Web Servers, SSL & Security Headers
## Day 105: Enable HTTPS in Nginx

## Objective
Harden HTTPS configuration in Nginx beyond basic certificate installation. Restrict TLS versions, enforce strong ciphers, and configure SSL session caching.

## Environment
- OS: Ubuntu 24.04, Oh My Zsh
- Web Server: Nginx 1.24.0
- Certificates: mkcert

## Description
Added security-focused SSL directives to the existing HTTPS server block. Disabled outdated TLS 1.0 and 1.1, restricted cipher suites to modern strong ciphers only, enabled server-side cipher preference, and configured SSL session caching for performance. Verified that weak protocols are rejected.

## What Was Done
- Added `ssl_protocols TLSv1.2 TLSv1.3;` to disable TLS 1.0 and 1.1
- Configured `ssl_ciphers` with a list of strong cipher suites (GCM and ChaCha20 only)
- Enabled `ssl_prefer_server_ciphers on;` — server picks the best cipher
- Added `ssl_session_cache shared:SSL:10m;` and `ssl_session_timeout 10m;` for performance
- Tested TLS 1.0, 1.1, 1.2, and 1.3 connections — only 1.2 and 1.3 work
- Verified cipher suites with `openssl s_client`

## Commands Used
- `sudo nano /etc/nginx/sites-available/proxy.local`
- `sudo nginx -t && sudo systemctl reload nginx`
- `curl -k --tlsv1.2 https://proxy.local:8443/ -o /dev/null -w "%{http_code}\n"`
- `curl -k --tlsv1.3 https://proxy.local:8443/ -o /dev/null -w "%{http_code}\n"`
- `curl -k --tlsv1.0 https://proxy.local:8443/ -o /dev/null -w "%{http_code}\n"`
- `curl -k --tlsv1.1 https://proxy.local:8443/ -o /dev/null -w "%{http_code}\n"`
- `echo | openssl s_client -connect proxy.local:8443 -tls1_2 2>/dev/null | grep Cipher`
- `echo | openssl s_client -connect proxy.local:8443 -tls1_3 2>/dev/null | grep Cipher`

## Verification
- `curl --tlsv1.2` returned `200` — TLS 1.2 works
- `curl --tlsv1.3` returned `200` — TLS 1.3 works
- `curl --tlsv1.0` returned `35` (error) — TLS 1.0 rejected
- `curl --tlsv1.1` returned `35` (error) — TLS 1.1 rejected
- `openssl s_client` showed strong ciphers: `ECDHE-RSA-AES256-GCM-SHA384` and `TLS_AES_256_GCM_SHA384`

## Issues Encountered
No issues.

## Key Concept
**TLS hardening.** HTTPS alone does not guarantee security. Weak TLS versions (1.0, 1.1) and weak ciphers (RC4, DES, 3DES, CBC) must be explicitly disabled. `ssl_protocols` restricts TLS versions. `ssl_ciphers` defines which cipher suites are allowed. `ssl_prefer_server_ciphers on` ensures the server, not the client, chooses the strongest cipher. SSL session caching improves performance for returning clients by reusing session parameters.

## Takeaways
1. Always disable TLS 1.0 and 1.1 — they are vulnerable to POODLE and BEAST.
2. Use only AEAD ciphers (GCM, ChaCha20) — avoid CBC ciphers.
3. `ssl_prefer_server_ciphers on` prevents downgrade attacks.
4. SSL session caching reduces CPU overhead for repeated connections.
5. Test with `curl --tlsv1.X` to verify which protocols are accepted.
