# Stage 01: Linux & Web Servers, SSL & Security Headers
## Day 114: HTTP Method Limiting & Request Body Size

## Objective
Restrict HTTP methods to only those required by the application and limit request body size to prevent abuse and resource exhaustion.

## Environment
- OS: Ubuntu 24.04, Oh My Zsh
- Web Server: Nginx 1.24.0

## Description
Added `limit_except` directives to block all HTTP methods except GET and HEAD for static content and API endpoints. Set `client_max_body_size` to 0 for static content and 1 MB for API requests. Verified that blocked methods return 403 and oversized requests return 413.

## What Was Done
- Tested that POST was allowed before the change
- Added `limit_except GET HEAD { deny all; }` to the static `/` location
- Added `limit_except GET { deny all; }` to the `/api/` location
- Set `client_max_body_size 0;` for static content
- Set `client_max_body_size 1m;` for API requests
- Verified GET and HEAD still work
- Verified POST, PUT, DELETE return 403
- Verified large request bodies return 413

## Commands Used
- `curl -k -X POST https://secure.local:8443/ -o /dev/null -w "%{http_code}\n"`
- `sudo nano /etc/nginx/sites-available/secure.local`
- `sudo nginx -t && sudo systemctl reload nginx`
- `curl -k -X PUT https://secure.local:8443/ -o /dev/null -w "%{http_code}\n"`
- `curl -k -X DELETE https://secure.local:8443/api/status -o /dev/null -w "%{http_code}\n"`
- `dd if=/dev/zero bs=1M count=2 2>/dev/null | curl -k -X POST https://secure.local:8443/api/status -H "Content-Type: application/octet-stream" --data-binary @- -o /dev/null -w "%{http_code}\n"`

## Verification
- GET on static and API returned 200
- HEAD on static returned 200
- POST, PUT, DELETE on static returned 403
- POST, PUT, DELETE on API returned 403
- 2 MB body on API returned 413 Request Entity Too Large
- `sudo nginx -t` passed

## Issues Encountered
No issues.

## Key Concept
**Method restriction and body size limits.** Every open HTTP method is part of the attack surface. Static sites only need GET and HEAD. APIs typically need GET and POST. Methods like PUT, DELETE, PATCH, and OPTIONS should be blocked unless explicitly required. `limit_except` whitelists allowed methods and denies all others with a 403. `client_max_body_size` prevents denial-of-service attacks via large request bodies. For static content, even 0 is acceptable (no body at all). For APIs, a reasonable limit like 1 MB prevents abuse while allowing legitimate requests.

## Takeaways
1. `limit_except` is the simplest way to whitelist HTTP methods in Nginx.
2. 403 is the correct response for denied methods — "Forbidden".
3. `client_max_body_size 0` completely disables request bodies for static content.
4. Always set a body size limit for API endpoints to prevent resource exhaustion.
5. Method restriction and size limits are the last line of defense before the application.
