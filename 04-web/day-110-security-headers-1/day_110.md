# Stage 01: Linux & Web Servers, SSL & Security Headers
## Day 110: X-Frame-Options & X-Content-Type-Options

## Objective
Protect the website against clickjacking and MIME-sniffing attacks by adding two HTTP security headers.

## Environment
- OS: Ubuntu 24.04, Oh My Zsh
- Web Server: Nginx 1.24.0

## Description
Added `X-Frame-Options: DENY` to prevent the site from being embedded in iframes (clickjacking protection). Added `X-Content-Type-Options: nosniff` to prevent browsers from guessing MIME types (MIME-sniffing protection). Verified headers are present on both 200 and 404 responses.

## What Was Done
- Checked that no security headers were present before the change
- Added `X-Frame-Options "DENY" always;` to the Nginx server block
- Added `X-Content-Type-Options "nosniff" always;` to the Nginx server block
- Added the same headers to the `/api/` location block (required due to `proxy_pass`)
- Verified headers appear on 200, 404, and proxied responses
- Tested clickjacking protection by creating an HTML file with an iframe

## Commands Used
- `curl -I -k https://secure.local:8443/ | grep -i "x-frame\|x-content"`
- `sudo nano /etc/nginx/sites-available/secure.local`
- `sudo nginx -t && sudo systemctl reload nginx`
- `curl -I -k https://secure.local:8443/nonexistent`
- `curl -I -k https://secure.local:8443/api/status`

## Verification
- `curl -I -k https://secure.local:8443/` returned `X-Frame-Options: DENY` and `X-Content-Type-Options: nosniff`
- Headers were present even on 404 responses (due to `always`)
- Headers were present on `/api/` proxied responses
- iframe test in browser showed blocked content

## Issues Encountered
1. **Headers missing on `/api/` route** — `add_header` in the `server` block is not inherited by `location` blocks with `proxy_pass`. Fixed by duplicating headers inside the `/api/` location.

## Key Concept
**Clickjacking and MIME-sniffing.** Clickjacking tricks users into clicking on something different from what they see by embedding a site in an invisible iframe. `X-Frame-Options` tells the browser never to display the site in a frame. MIME-sniffing allows browsers to guess the content type of a file, which can turn a harmless text file into an executable script. `X-Content-Type-Options: nosniff` forces the browser to respect the server's `Content-Type` header. Both headers are simple but critical defenses.

## Takeaways
1. `X-Frame-Options: DENY` is the strictest clickjacking protection.
2. `X-Content-Type-Options: nosniff` prevents MIME-based attacks.
3. `always` ensures headers are sent on error responses too.
4. `proxy_pass` blocks require headers to be duplicated — they do not inherit from the parent.
5. Two lines of config can prevent two classes of attacks.
