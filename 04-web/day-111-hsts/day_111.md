# Stage 01: Linux & Web Servers, SSL & Security Headers
## Day 111: Strict-Transport-Security (HSTS)

## Objective
Enable HTTP Strict Transport Security to force browsers to always use HTTPS, preventing SSL-stripping attacks.

## Environment
- OS: Ubuntu 24.04, Oh My Zsh
- Web Server: Nginx 1.24.0

## Description
Added the `Strict-Transport-Security` header (HSTS) with a 1-year max-age and `includeSubDomains` directive. Verified the header is present on HTTPS responses. Confirmed that the HTTP-to-HTTPS redirect still functions for first-time visitors while HSTS protects returning visitors.

## What Was Done
- Verified no HSTS header was present before the change
- Added `Strict-Transport-Security "max-age=31536000; includeSubDomains" always;` to the server block
- Added the same header to the `/api/` location block
- Verified the header appears on HTTPS responses
- Confirmed HTTP redirect still works alongside HSTS
- Checked HSTS status in browser DevTools (Application → Security)

## Commands Used
- `curl -I -k https://secure.local:8443/ 2>/dev/null | grep -i strict`
- `sudo nano /etc/nginx/sites-available/secure.local`
- `sudo nginx -t && sudo systemctl reload nginx`
- `curl -I http://secure.local:8080/`

## Verification
- `curl -I -k https://secure.local:8443/` returned `Strict-Transport-Security: max-age=31536000; includeSubDomains`
- `curl -I http://secure.local:8080/` still returned 301 redirect to HTTPS
- Both static and API routes included the HSTS header

## Issues Encountered
No issues.

## Key Concept
**HSTS vs redirects.** A 301 redirect only works after the first HTTP request has already been sent in plaintext. An attacker can intercept this first request (SSL-stripping). HSTS tells the browser: "Remember — this site is HTTPS-only for the next N seconds." On subsequent visits, the browser internally rewrites `http://` to `https://` before sending any request. `includeSubDomains` extends this to all subdomains. `preload` (not used here) allows the site to be hardcoded into browser HSTS preload lists.

## Takeaways
1. HSTS prevents SSL-stripping by forcing HTTPS at the browser level.
2. `max-age=31536000` = 1 year in seconds.
3. `includeSubDomains` protects all subdomains of the site.
4. HSTS complements, not replaces, HTTP-to-HTTPS redirects.
5. First visit still relies on the redirect; subsequent visits use HSTS.
