# Stage 01: Linux & Web Servers, SSL & Security Headers
## Day 115: [FINAL LAB] Complete Secure Website

## Objective
Deploy a fully secured production-grade website from scratch, integrating all skills from Month 4: static site serving, reverse proxy, HTTPS, redirects, security headers, and method restrictions.

## Environment
- OS: Ubuntu 24.04, Oh My Zsh
- Web Server: Nginx 1.24.0
- Application: Python 3.12, Flask 3.1
- Certificates: mkcert

## Description
Deployed a complete secure website (`shield.local`) combining static content and a Flask API behind Nginx. Configured HTTPS with strong ciphers and TLS 1.2/1.3 only, HTTP-to-HTTPS and www-to-non-www redirects, all six security headers, method restrictions, and request body size limits. Passed 10 of 12 checklist items; remaining two are understood and fixable.

## What Was Done
- Created folder structure and deployed static HTML/CSS (Shield theme)
- Wrote Flask app with `/api/health` and `/api/info` endpoints
- Generated mkcert certificate for `shield.local` and `www.shield.local`
- Wrote Nginx config with three server blocks: www redirect, HTTP redirect, HTTPS main site
- Configured TLS 1.2/1.3 only with GCM/ChaCha20 ciphers
- Added all 6 security headers to both server block and `/api/` location
- Restricted methods to GET and HEAD, set body size limits
- Configured separate access and error logs
- Tested against 12-point checklist

## Commands Used
- `sudo mkdir -p /var/www/shield.local/css`
- `sudo cp /var/www/secure.local/* /var/www/shield.local/`
- `python3 -m venv venv && source venv/bin/activate && pip install flask`
- `python3 app.py 5300`
- `mkcert -cert-file shield.local.pem -key-file shield.local-key.pem shield.local www.shield.local`
- `sudo tee /etc/nginx/sites-available/shield.local`
- `sudo ln -s /etc/nginx/sites-available/shield.local /etc/nginx/sites-enabled/`
- `echo "127.0.0.1 shield.local www.shield.local" | sudo tee -a /etc/hosts`
- `sudo nginx -t && sudo systemctl reload nginx`
- `curl -I http://shield.local:8082/`
- `curl -k https://shield.local:8443/`
- `curl -k https://shield.local:8443/api/health`

## Issues Encountered
1. **502 Bad Gateway on all routes** — `server_name` was still `secure.local` after copying the config. Fixed by updating to `shield.local`.
2. **API /health returned 404** — Flask app had `/api/status` but not `/api/health`. Needs a new route decorator.
3. **TLS 1.0 not blocked** — likely a conflicting default config listening on the same port. Fix by removing the default symlink.

## Key Concept
**Defense in depth.** This lab demonstrates layered security: HTTPS encrypts traffic, security headers protect the browser, method restrictions limit the attack surface, and body size limits prevent resource exhaustion. Each layer addresses a different class of attacks. No single configuration line makes a site secure — the combination of all protections working together does. This is the core DevSecOps principle: security is not a feature, it's a property of the entire system.

## Takeaways
1. Copying configs requires careful review — check domain, paths, ports.
2. All 6 security headers together provide strong browser-side protection.
3. Method restrictions and body size limits are the last line before the application.
4. A 12-point checklist catches issues before they reach production.
5. Month 4 complete: from zero to a fully secured Nginx + Flask deployment.
