# Stage 01: Python & API
## Day 127: [LAB] File Permissions Audit Script

## Objective
Build a real DevSecOps tool that scans a directory for dangerous file permissions and saves a JSON report.

## Environment
- OS: Ubuntu 24.04, Oh My Zsh
- Python: 3.12
- Tools: venv, VSCode

## Description
Built a file permissions auditor that scans a target directory, checks Unix permissions for each file, flags 777 as WARNING, and saves a structured JSON report. Simulates a real security audit script used in DevSecOps pipelines.

## What Was Done
- Wrote `audit_folder()` using `os.listdir()` and `os.stat()`
- Extracted Unix permissions with `oct(info.st_mode)[-3:]`
- Applied WARNING/OK status using ternary operator
- Logged each file at appropriate level — WARNING or INFO
- Wrote `save_report()` to persist audit results as JSON
- Tested with a `chmod 777` file to verify WARNING detection

## Commands Used
- `python3 permission_check.py`
- `touch test && chmod 777 test`
- `cat report.json`

## Verification
- 644 files logged as INFO with OK status ✅
- 777 file logged as WARNING correctly ✅
- report.json contains all scanned files with mode and status ✅

## Issues Encountered
1. **report.json saved to wrong directory** — ran script from repo root, report created there. Fixed by running from day folder.
2. **return inside loop** — placed `return report` inside the for loop, only first file was processed. Fixed by dedenting outside the loop.

## Key Concepts
- `os.listdir()` — returns list of filenames in directory
- `os.stat()` — returns file metadata including permissions
- `oct(info.st_mode)[-3:]` — extracts Unix permission string (e.g. "644", "777")
- `json.dump(indent=4)` — saves human-readable JSON report

## Takeaways
1. `oct()` converts raw st_mode integer to readable Unix permissions
2. Ternary operator `"WARNING" if condition else "OK"` keeps status logic concise
3. return placement relative to loop indentation is critical — always double-check
4. JSON reports are standard output format for security audit scripts
