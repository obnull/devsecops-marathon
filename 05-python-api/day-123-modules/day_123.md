# Stage 01: Python & API
## Day 123: Modules, Imports, __name__ == "__main__"

## Objective
Learn to structure Python projects with modules and packages. Understand import system and __name__ guard.

## Environment
- OS: Ubuntu 24.04, Oh My Zsh
- Python: 3.12
- Tools: venv, VSCode

## Description
Built a modular Host Monitor utility split across multiple files and packages. Core logic separated into reusable modules under utils/ package.

## Project Structure
day-123-modules/

├── main.py

├── checks.py

└── utils/

├── init.py

├── network.py

└── formatters.py
## What Was Done
- Created `utils/` package with `__init__.py`
- Wrote `network.py` with `simulate_ping()` and `is_port_in_range()`
- Wrote `formatters.py` with `format_table()` and `severity_color()`
- Wrote `checks.py` with `check_http_status()` and `check_resource_usage()`
- Imported modules using `from utils.network import ...` syntax
- Used `if __name__ == "__main__"` guard in main.py

## Commands Used
- `python3 main.py`

- Network simulation runs for all hosts ✅
- HTTP status check returns healthy/unhealthy correctly ✅
- Resource severity calculated and color-coded ✅
- Port validation correctly rejects 99999 ✅
- `__name__` prints `__main__` on direct run ✅

## Issues Encountered
None.

## Key Concepts
- Package = directory with `__init__.py`
- `from module import func` — import specific function
- `__name__ == "__main__"` — guard prevents code running on import
- Separating concerns into modules makes code reusable and testable

## Takeaways
1. Always split large scripts into modules by responsibility
2. `__init__.py` makes a directory a Python package
3. `if __name__ == "__main__"` is mandatory in every entry point script
