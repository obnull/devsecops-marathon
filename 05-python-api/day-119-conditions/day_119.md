# Stage 01: Python & API
## Day 119: Conditions, Boolean Logic

## Objective
Master decision-making logic for monitoring, alerting, and security checks.

## Environment
- OS: Ubuntu 24.04, Oh My Zsh
- Python: 3.12
- Tools: venv (shared from Day 116)

## Description
Built two scripts: HTTP status checker with alert logic and file permission security auditor. Used compound conditions, isinstance() checks, and octal permission masks.

## What Was Done
- Wrote http_status_checker.py — status classification, alert determination
- Wrote permission_checker.py — security policy validation for file permissions
- Implemented multi-condition alerts (status >= 500 OR response_time > 5s)
- Used isinstance() for type validation
- Applied octal masks (0o644, 0o777) for permission checks
- Detected secrets files with insecure permissions

## Commands Used
- `python3 src/http_status_checker.py`
- `python3 src/permission_checker.py`

## Verification
- Status classification: 2xx/3xx/4xx/5xx correct
- Alert triggers on 500 + slow response
- .env (644) flagged as secrets violation
- 777 permissions detected as CRITICAL
- Config files with group-writable caught

## Issues Encountered
None.

## Key Concept
Conditions transform scripts from displaying data to making decisions. Status 500 isn't just output — it's a decision point: alert? who? when?

## Takeaways
1. if/elif/else covers all branches
2. Compound conditions (and/or) enable precise alerting
3. Octal masks handle Linux permissions in Python
4. Security auditing automatable with simple conditional checks
