# Stage 01: Python & API
## Day 131: Parsing JSON and XML Responses

## Objective
Learn to extract specific fields from JSON API responses by iterating over lists of objects.

## Environment
- OS: Ubuntu 24.04, Oh My Zsh
- Python: 3.12
- Tools: venv, VSCode

## Description
Built a GitHub repository lister that parses JSON API responses. Script fetches all public repos for a given username, iterates over the list of repo objects, and extracts name and language fields from each.

## What Was Done
- Wrote `get_repos()` making GET request to `/users/{username}/repos`
- Used f-string to interpolate username into URL
- Parsed JSON response as list of dictionaries with `r.json()`
- Iterated over list and extracted `name` and `language` fields
- Logged status code for each request

## Commands Used
- `python3 main.py`

## Verification
- Request returns 200 and list of repositories ✅
- name and language printed for each repo ✅
- f-string correctly interpolates username into URL ✅

## Issues Encountered
1. **Missing f-string in URL** — wrote `"https://api.github.com/users/{username}/repos"` without `f` prefix. Username not substituted into URL. Fixed by adding `f`.
2. **print(get_repos) instead of get_repos()** — printed function object instead of calling it. Fixed by adding parentheses.
3. **WSL DNS failure** — connection timeout fixed with `echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf`.

## Key Concepts
- `r.json()` returns Python object — dict for single object, list for collections
- GitHub `/repos` endpoint returns a list of repo objects
- Access nested fields with `item["name"]`, `item["language"]`
- Iterate over JSON arrays with standard `for item in data`

## Takeaways
1. Always check if API returns a dict or a list before accessing fields
2. f-string prefix is required for variable interpolation in URLs
3. Calling a function and referencing a function are different — always use ()
4. language field can be None if repo has no detected language
