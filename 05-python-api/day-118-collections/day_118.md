# Stage 01: Python & API
## Day 118: Lists, Tuples, Dicts, Sets

## Objective
Master Python collections using real DevOps data: API responses, logs, configs.

## Environment
- OS: Ubuntu 24.04, Oh My Zsh
- Python: 3.12
- Tools: venv (shared from Day 116)

## Description
Practiced all four collection types in DevOps scenarios. Dicts for API responses, lists for IPs, sets for deduplication, tuples for immutable configs.

## What Was Done
- Wrote health_check.py — dict-based API simulation with health analysis
- Wrote log_analyzer.py — IP extraction, unique IPs via set, request counting
- Used dict.get() for safe key access
- Used set() for unique IPs
- Implemented per-IP statistics with nested dicts

## Commands Used
- `python3 src/health_check.py`
- `python3 src/log_analyzer.py`

## Verification
- Health check identifies degraded services
- 14 total requests, 4 unique IPs
- Per-IP counts correct
- Suspicious activity detection works

## Issues Encountered
None.

## Key Concept
90% of DevOps data structures are dicts (JSON APIs) and lists (servers, ports). Sets for deduplication. Tuples for constants.

## Takeaways
1. dict.get(key, default) safer than dict[key]
2. set() instantly deduplicates
3. Nested dicts model real API responses
