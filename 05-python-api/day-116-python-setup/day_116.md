# Stage 01: Python & API
## Day 116: Python Setup (venv, pip, VSCode)

## Objective
Set up Python development environment with virtual environment, package manager, and IDE configuration.

## Environment
- OS: Ubuntu 24.04, Oh My Zsh
- Python: 3.12
- Tools: venv, pip, VSCode

## Description
Created isolated Python environment for Month 5. Configured venv, installed core packages, set up .gitignore.

## What Was Done
- Created venv with `python3 -m venv venv`
- Activated and verified isolation with `which python3`
- Wrote hello_devsecops.py verification script
- Installed requests and python-dotenv
- Saved dependencies to requirements.txt
- Created .gitignore for Python projects

## Commands Used
- `python3 -m venv venv`
- `source venv/bin/activate`
- `pip install --upgrade pip`
- `pip install requests python-dotenv`
- `pip freeze > requirements.txt`

## Verification
- Script detects venv status correctly 
- `which python3` points to venv path 
- requirements.txt has correct packages 

## Issues Encountered
1. **pip upgrade outside venv** — running without activated venv updates system pip. Fixed by sourcing venv first.

## Key Concept
Virtual environments isolate project dependencies. Always work inside venv, never install packages globally.

## Takeaways
1. Activate venv before any pip operations
2. requirements.txt tracks exact dependency versions
3. .gitignore must exclude venv/ and .env
