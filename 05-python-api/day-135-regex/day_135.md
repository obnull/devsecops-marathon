## Issues Encountered

2. **Unnecessary append logic** — tried to build result list manually with `append`. Replaced with direct `return re.findall(...)` since findall already returns a list.

## Key Concepts
- `re.findall(pattern, string)` — returns list of all matches, not just the first
- `\d{1,3}` — matches 1 to 3 digits, covers all valid IP octets
- `\.` — escaped dot matches literal `.`, not any character
- `https?` — `?` makes preceding character optional, matches both http and https
- `\S+` — one or more non-whitespace characters, good for URLs

## Takeaways
1. Use `findall` when you need all matches, `search` when you need just the first
2. Always use raw strings `r""` for regex — avoids backslash interpretation issues
3. `findall` returns strings directly — no need to call `.group()`
4. `\S+` is a quick URL catcher but may grab trailing punctuation — good enough for logs
