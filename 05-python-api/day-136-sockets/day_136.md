# Stage 01: Python & API
## Day 136: Socket Programming Basics

## Objective
Learn to check port availability on remote hosts using Python's socket module — foundation for the final port scanner project.

## Environment
- OS: macOS (Apple Silicon)
- Python: 3.9.6
- Tools: venv, VSCode

## Description
Built a port checker using raw TCP sockets. No external libraries — just the standard socket module. The function tries to establish a connection and returns True or False based on whether the port responds. Clean, minimal, reusable.

## What Was Done
- Wrote `check_port(host, port)` using `socket.socket()` with TCP settings
- Set 1 second timeout to avoid hanging on closed ports
- Used `connect_ex()` which returns 0 on success instead of raising an exception
- Closed socket after each check
- Tested against google.com on ports 80, 443, 8080 — all open

## Commands Used
- `python3 main.py`

## Verification
- Port 80 (HTTP) open on google.com ✅
- Port 443 (HTTPS) open on google.com ✅
- Port 8080 open on google.com ✅

## Issues Encountered
1. **Typo in settimeout** — wrote `stetimeout()` instead of `settimeout()`. Fixed before running.
2. **Arguments passed as single string** — wrote `connect_ex(("google.com, 80"))` with both values inside one string. Fixed to `connect_ex(("google.com", 80))` — a proper tuple with two elements.
3. **return outside function** — wrote return statements at module level. Fixed by wrapping everything in `def check_port()`.

## Key Concepts
- `socket.AF_INET` — IPv4 addressing
- `socket.SOCK_STREAM` — TCP protocol
- `settimeout(1)` — prevents indefinite hang on unresponsive ports
- `connect_ex()` — returns error code instead of raising exception; 0 means success
- Always close the socket after use — it's a system resource

## Takeaways
1. Create a new socket inside the function — not globally. Each check needs a fresh connection
2. `connect_ex()` is cleaner than `connect()` for port scanning — no try/except needed
3. `settimeout()` is essential — without it the scanner hangs on every closed port
4. `return result == 0` is more Pythonic than `if result == 0: return True`
