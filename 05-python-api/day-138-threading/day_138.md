# Stage 01: Python & API
## Day 138: Threading — Parallel Tasks & Thread Arguments

## Objective
Understand how Python threads work and learn how to run multiple tasks in parallel by creating, starting, and synchronizing threads.

## Environment
- OS: macOS
- Python: 3.9.6
- Tools: venv, VSCode

## Description
Learned the basics of Python threading and built a small multithreaded task runner. Created multiple threads dynamically from a list of tasks, passed arguments into threads using `args`, started threads in parallel, and waited for completion with `join()`.

## What Was Done
- Created threads using `threading.Thread`
- Learned the difference between passing a function reference and calling a function immediately
- Used `target` to define the function executed by a thread
- Used `start()` to launch threads
- Used `join()` to wait for thread completion
- Stored created threads in a list for further management
- Created threads dynamically inside a loop
- Passed task-specific values into threads using `args=(value,)`
- Learned why `args` requires a tuple format, including the single-element tuple syntax

## Commands Used
- `python3 main.py`

## Verification
- Multiple threads started successfully ✅
- Each thread received its own argument value ✅
- Main thread waited for worker threads using `join()` ✅
- Thread execution order changed between runs because scheduling is controlled by the operating system ✅

## Issues Encountered
1. **Using `target=function()` instead of `target=function`** — called the function immediately instead of passing it to the thread. Fixed by passing the function reference without parentheses.
2. **Incorrect `append()` usage** — used `threads = threads.append(thread)`. `append()` modifies the list directly and returns `None`. Fixed by using `threads.append(thread)`.
3. **Resetting the thread list inside the loop** — recreated `threads` on every iteration, removing previously created threads. Fixed by creating the list before the loop.
4. **Incorrect `args` format** — passed multiple values instead of the current task value. Fixed by using `args=(number,)`.
5. **Missing tuple comma** — learned that `(value)` is not a tuple, while `(value,)` is a single-element tuple.

## Key Concepts
- `threading.Thread` creates a new execution thread
- `target` defines which function the thread will execute
- Function references are passed without `()`
- `start()` begins thread execution
- `join()` blocks the main thread until the worker finishes
- Thread objects can be stored in lists and managed together
- `args` passes arguments to the target function when the thread starts
- Single-value tuples require a trailing comma: `(value,)`
- Thread execution order is not guaranteed because the OS scheduler controls execution

## Takeaways
1. Threads should be created dynamically instead of manually writing separate variables
2. Lists make it easier to manage multiple threads
3. `target` and `args` separate the task from the data required to execute it
4. `join()` is required when the main program must wait for background work
5. Multithreading is useful for I/O-bound tasks such as network checks and service monitoring
