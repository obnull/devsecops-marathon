# Stage 01: Python & API
## Day 130: API Interaction — Authentication, Tokens

## Objective
Learn to authenticate with REST APIs using Bearer tokens and request headers.

## Environment
- OS: Ubuntu 24.04, Oh My Zsh
- Python: 3.12
- Tools: venv, VSCode

## Description
Built a GitHub API client with Bearer token authentication. Script sends authenticated GET request to /user endpoint and returns full profile data. Covers header construction, token passing, and authenticated vs unauthenticated API access.

## What Was Done
- Generated GitHub Personal Access Token with repo and user scopes
- Constructed Authorization header with Bearer token
- Wrote `get_user_info()` making authenticated request to `/user` endpoint
- Compared authenticated vs unauthenticated responses
- Replaced token with placeholder before commit

## Commands Used
- `python3 main.py`

## Verification
- Unauthenticated request to `/users/obnull` returns public profile ✅
- Authenticated request to `/user` returns full private profile ✅
- 401 returned when token missing or malformed ✅
- 200 returned with valid token ✅

## Issues Encountered
1. **Typo in header key** — wrote `"Autorization"` missing letter h. API returned 401. Fixed by correcting spelling to `"Authorization"`.
2. **Wrong endpoint** — used `/users/obnull` instead of `/user` for authenticated request. Fixed to `/user` which GitHub resolves from token.
3. **Token exposed in chat** — accidentally shared real GitHub PAT. Immediately revoked on github.com/settings/tokens and generated new one.
4. **WSL DNS failure** — connection timeout fixed with `echo "nameserver 8.8.8.8" | sudo tee /etc/resolv.conf`.

## Key Concepts
- Bearer token passed in Authorization header: `"Bearer YOUR_TOKEN"`
- `/user` endpoint returns data for the authenticated user
- 401 Unauthorized — token missing, wrong, or malformed
- Never hardcode real tokens in source code — use environment variables

## Takeaways
1. Authorization header spelling must be exact — one typo returns 401
2. `/user` vs `/users/{username}` — authenticated vs public endpoint
3. Always revoke exposed tokens immediately
4. Store tokens in .env files, never in source code — Day 132 covers this
