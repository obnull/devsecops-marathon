Turn Python scripts into proper CLI tools that accept arguments at runtime instead of hardcoded values.

## Description
Wrapped the Day 136 port checker in a CLI interface using argparse. Script now accepts host and ports as command-line arguments — no touching the code to change targets. Ran it against google.com with three ports and got clean output.

## What Was Done
- Added `argparse.ArgumentParser` with description
- Added `--host` as required string argument
- Added `--ports` as required list of integers using `nargs="+"`
- Replaced hardcoded values in `__main__` with `args.host` and `args.ports`
- Tested with `--help` flag — argparse generated usage automatically

## Commands Used
- `python3 main.py --host google.com --ports 80 443 8080`
- `python3 main.py --help`

## Verification
- Port 80 open on google.com ✅
- Port 443 open on google.com ✅
- Port 8080 open on google.com ✅
- `--help` shows correct usage and argument descriptions ✅

## Issues Encountered
1. **parser inside function** — put `ArgumentParser` setup inside `check_port()`. Parser must live at module level, outside any function. Fixed by moving it out.
2. **`required-True` typo** — wrote dash instead of equals. Fixed to `required=True`.
3. **`--port` instead of `--ports`** — mismatched argument name caused `AttributeError: Namespace has no attribute 'port'`. Fixed to `--ports` everywhere.
4. **`args.port` in print** — used wrong attribute name in f-string. Fixed to `args.host`.

## Key Concepts
- `ArgumentParser` parses `sys.argv` automatically — no manual string splitting
- `nargs="+"` — accepts one or more values, returns a list
- `type=int` — converts string input to integer automatically
- `required=True` — argparse exits with error message if argument is missing
- `args.host`, `args.ports` — access parsed values as object attributes

## Takeaways
1. Put parser setup at module level, not inside functions
2. `nargs="+"` is the standard pattern for accepting lists from CLI
3. Argument name in `add_argument` must match attribute access — `--ports` → `args.ports`
4. argparse gives you `--help` for free — always add `help=` descriptions
