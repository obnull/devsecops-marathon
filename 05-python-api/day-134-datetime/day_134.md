## Objective
Learn to work with timestamps, date arithmetic, and file modification times for DevOps tooling.

## Description
Built three utility functions around datetime. Nothing fancy — timestamp formatter, a logger wrapper, and a file freshness checker. The last one turned out useful: checks whether a file was touched within N hours, which is handy for monitoring stale configs or stale reports.

## What Was Done
- Wrote `get_timestamp()` returning current time as formatted string
- Wrote `log_with_time()` wrapping timestamp in brackets and printing with message
- Wrote `is_recent()` using `os.path.getmtime()` and `timedelta` to check file age
- Tested all three in `__main__` block against `main.py` itself

## Commands Used
- `python3 main.py`

## Verification
- `get_timestamp()` returns correctly formatted string ✅
- `log_with_time()` prints `[timestamp] message` format ✅
- `is_recent()` returns True for recently modified file ✅

## Issues Encountered
1. `datetime.now` without parentheses — referenced the method instead of calling it. Caught before running, fixed immediately.
2. `FileNotFoundError` on `is_recent("main.py")` — ran script from repo root, relative path didn't resolve. Fixed by running from the day folder.

## Key Concepts
- `datetime.now()` — current local time as datetime object
- `strftime()` — formats datetime into any string you need
- `timedelta` — arithmetic on dates: add/subtract hours, days, weeks
- `os.path.getmtime()` — returns file's last modification time as Unix timestamp
- `datetime.fromtimestamp()` — converts Unix timestamp back to datetime object

## Takeaways
1. Always call `datetime.now()` with parentheses — without them you get the method, not the value
2. `timedelta` makes date math readable: `now - timedelta(hours=24)` is cleaner than manual seconds
3. Relative paths depend on where you run the script from — use absolute paths or `__file__` for reliability
