# Stage 01: Python & API
## Day 125: Exception Handling (try/except/finally)

## Objective
Learn to handle runtime errors gracefully in DevOps scripts using try/except blocks.

## Environment
- OS: Ubuntu 24.04, Oh My Zsh
- Python: 3.12
- Tools: venv, VSCode

## Description
Built a config reader and parser with proper exception handling. Script catches specific errors instead of crashing, returns None as error signal between functions, and handles edge cases like empty lines.

## What Was Done
- Wrote `read_config()` with try/except for FileNotFoundError and PermissionError
- Wrote `parse_config()` with None guard and empty line skip
- Used `is None` instead of `== None` for Pythonic comparison
- Tested both success and failure scenarios

## Commands Used
- `python3 main.py`
- `echo "host=localhost\nport=8080\nuser=admin" > config.txt`

## Verification
- Missing file returns None and prints error message ✅
- Valid config parsed into dictionary correctly ✅
- Empty lines skipped without IndexError ✅
- None input returns empty dict ✅

## Issues Encountered
1. **return inside loop** — placed `return result` inside the for loop, function returned after first line only. Fixed by dedenting return outside the loop.
2. **IndexError on empty line** — config.txt had trailing newline, `split("=")` on empty string produced single-element list. Fixed with `if not line: continue`.

## Key Concepts
- `try/except` catches specific exceptions without crashing the script
- `return None` is a clean error signal between functions
- `is None` is preferred over `== None` in Python
- Each except block should handle one specific error type

## Takeaways
1. Always catch specific exceptions, not bare except
2. Use None as error signal between functions
3. return inside a loop exits after first iteration — always check indentation
4. Defensive checks (if not line) prevent unexpected crashes on real data
