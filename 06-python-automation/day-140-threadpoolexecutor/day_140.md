# Stage 01: Python Automation

## Day 140: ThreadPoolExecutor

## Objective

Learning how to execute tasks concurrently using ThreadPoolExecutor and manage results with Future objects.

## Environment

Ubuntu 24.04, Python 3.x
VSCode, venv

## Description

Replaced manual thread management with ThreadPoolExecutor. Learned how to create a worker pool, submit tasks, collect Future objects and retrieve execution results. Built a small parallel file processing example.

## What Was Done

* Learned difference between `threading.Thread` and ThreadPoolExecutor
* Created a worker pool with limited number of threads
* Used `executor.submit()` to send tasks for execution
* Learned why functions are passed without `()`
* Worked with Future objects and retrieved results using `future.result()`
* Built parallel file size checker example

## Commands Used

* `python3 main.py`
* `python3 -m venv`
* `source venv/bin/activate`

## Verification

* ThreadPoolExecutor successfully created worker pool
* Multiple tasks were submitted and stored as Future objects
* Results were received using `future.result()`
* File processing logic worked with multiple input files

## Issues Encountered

1. Passed a list of files instead of individual file paths to `executor.submit()`.
   Fixed: processed each file separately inside a loop and created individual Future objects.

2. Confused `ThreadPoolExecutor()` arguments with task arguments.
   Fixed: understood that `max_workers` controls pool size, while `submit()` receives function and its data.

## Key Concept

ThreadPoolExecutor separates tasks from thread management:

`Executor → submit() → Future → result()`

`submit()` receives a function reference and arguments.
`Future` stores the state and result of an asynchronous task.

## Takeaways

1. ThreadPoolExecutor simplifies thread management
2. Number of workers and number of tasks are different things
3. Functions are passed without execution brackets
4. Future objects allow collecting results from background tasks
5. Thread pools are useful for automation tasks with many similar operations
