# Stage 01: Python Automation

## Day 142: Multiprocessing Basics

### Objective

Learn the basics of Python multiprocessing and understand the difference between threads and processes.

### Environment

* OS: Ubuntu
* Python: 3.9.6
* Editor: Visual Studio Code
* Module: `multiprocessing`

### Description

During this day, I studied how Python processes work and how they differ from threads.

Threads run inside the same process and share memory, while separate processes have isolated memory spaces.

I also learned how to create and start a child process using the `multiprocessing.Process` class.

### What Was Done

* Imported the `multiprocessing` module.
* Created a worker function.
* Created a child process with `multiprocessing.Process`.
* Passed the worker function using the `target` argument.
* Started the process using `start()`.
* Waited for the process to finish using `join()`.
* Tested how global variables behave in separate processes.
* Learned why processes do not share normal global variables.
* Used the `if __name__ == "__main__":` protection.
* Learned why this protection is especially important when using multiprocessing on Windows.

### Code Example

```python
import multiprocessing

counter = 0


def worker():
    global counter
    counter = counter + 1
    print(f"Worker counter: {counter}")


if __name__ == "__main__":
    process = multiprocessing.Process(target=worker)

    process.start()
    process.join()

    print(f"Main counter: {counter}")
```

### Commands Used

```bash
python3 main.py
```

### Verification

The child process received its own copy of the global variable.

Inside the child process, the counter was incremented:

```text
Worker counter: 1
```

The counter in the main process remained unchanged:

```text
Main counter: 0
```

This confirmed that the child process and the main process use separate memory spaces.

### Issues Encountered

At first, I created a new process inside the worker function.

This could cause recursive process creation because every new process would execute the same worker function and create another process.

The process creation logic was moved outside the worker function and placed inside:

```python
if __name__ == "__main__":
```

### Key Concept

Processes do not normally share memory.

Each process has its own independent memory space and its own copy of global variables.

The `if __name__ == "__main__":` condition prevents process creation code from running again when the Python module is loaded by a child process.

### Takeaways

* `multiprocessing.Process` creates a separate process.
* `target` specifies the function executed by the child process.
* `start()` launches the process.
* `join()` waits for the process to finish.
* Processes have isolated memory.
* Changing a global variable inside a child process does not change it in the main process.
* Process creation should be protected with `if __name__ == "__main__":`.
* Multiprocessing is useful for tasks that can benefit from running in separate processes.
