# Stage 01: Python Automation

## Day 141: Queue & Worker Threads

## Objective

Learned how to safely distribute tasks between multiple threads using a shared Queue.

## Environment

Ubuntu 24.04, Python 3.x
VSCode, venv

## Description

Built a simple producer-worker model using `queue.Queue` and `threading.Thread`. Learned how workers receive tasks from a shared queue, process them independently and terminate gracefully using sentinel values.

## What Was Done

* Created a shared task queue
* Added tasks using `put()`
* Retrieved tasks with `get()`
* Implemented a worker function with a `while` loop
* Used `None` as a sentinel value to stop workers
* Started multiple worker threads
* Waited for all threads to finish with `join()`

## Commands Used

* `python3 main.py`

## Verification

* All tasks were processed successfully
* Tasks were distributed between worker threads
* Every worker terminated after receiving a sentinel value
* Program exited without hanging

## Issues Encountered

1. Tried to iterate over `Queue` directly when creating threads.
   Fixed by creating a fixed number of worker threads with `range()`.

2. Checked for `None` before retrieving a task from the queue.
   Fixed by calling `get()` first, then checking the returned value.

## Key Concept

`Queue` provides thread-safe task distribution.

Workers continuously retrieve tasks using `get()`. A sentinel value (`None`) signals that no more work is available and allows each worker to exit its loop cleanly.

## Takeaways

1. `Queue` is designed for communication between threads
2. `put()` adds a task and `get()` removes the next task
3. Workers should process tasks in a loop
4. Each worker needs its own sentinel value to terminate
5. `join()` ensures all worker threads finish before the program exits
